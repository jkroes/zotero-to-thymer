The two duplicate fixes live at the 5 · upsert stage: searches are scoped to the configured workspace (node R1), and the index-lag grace covers renames as well as creates via titleSyncedAt (folded into the reachable? decision). Reachable → update in place (6a); unreachable → rebuild (6b). Annotations (7) run the same reachability + grace per annotation, nested under the reference's Annotations field tuple (created lazily, re-resolved if deleted). Warn-and-skip guards both a referenced field value (U5/U6) and a removed-but-referenced annotation (ANREF/ANRW) — both feed the "Synced with warnings" outcome.
